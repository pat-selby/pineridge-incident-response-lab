## Overview
This lab simulates an incident response scenario involving suspicious authentication
activity on a Linux system. The focus is on detection, evidence preservation,
documentation, and post-incident analysis.

## Scenario
Multiple failed login attempts were detected on a user account. Security personnel
were tasked with reviewing authentication logs, collecting evidence, and documenting
the incident.

## Objectives
- Detect suspicious login activity
- Preserve authentication logs as evidence
- Create an incident timeline
- Document actions taken
- Recommend preventative security controls

## Tools Used
- Ubuntu Linux (VM)
- System authentication logs
- Terminal utilities (grep, cp)
- Markdown documentation
